算是把这个博客搭起来了

  • CSS
  • Git
  • Javascript
learning

经过一番折腾总算把这个博客弄得差不多了，但还是有好多东西留下了遗憾。比如页面布局、整体架构感觉不是很喜欢想自己修改，苦于技术不到家，好多东西还没学过。哎，欲速则不达，以后在慢慢补齐就好了。这篇短文就当作测试了。

说下过程。首先需要安装 Node.js、Git、Python（别问我为什么安装 Python，不安装这个好多东西报错）等等编译环境，由于最近开始尝试使用 Github Atom，这个editor需要手工添加环境变量，于是顺便学习了一下环境变量的知识。

然后是安装 hexo，安装过程很慢，于是使用了淘宝的 npm 镜像 cnpm，然而还有好多诸如 independency 的概念没搞清楚，以后再慢慢学吧。

准备步骤算是完成了，接下来是按照 tommy351 的文档来进行配置工作。配置过程折腾时间最长了，因为不知道怎么操作，好多参数也不清楚，这位 tommy351 君的文档好多东西都没说清楚，真是难为我这个小白了，搜索了百度上很多前辈给出的建议才知道如何修改。

然后是部署的准备工作，修改完 _config.yml 里面的 deploy 参数之后，需要安装 hexo-deployer-git 这个插件。安装完之后，需要按照 Github Pages 的文档中的步骤来建立存放整个静态页面的库。完成之后，还需要生成一个 SSH 公钥，以便可以随时与 Github 进行连接，你可以从这里阅读官方文档，来了解如何 “Generating an SSH key”。然后你就可以通过命令行来连接到 Github 啦！

工作完成了，就可以部署到你所建立的库中了。然而，有一些页面元素，比如说搜索框（我使用的 yilia 主题把搜索框去掉了）、多说评论框的样式修改、页面样式等等，还有就是 .ejs （Embedded JavaScript） 模板库的使用方法，如何比较自动化地更新这个静态博客等等都要学习。鼓捣这个博客也算是个引子，让我了解了一些新技术。
